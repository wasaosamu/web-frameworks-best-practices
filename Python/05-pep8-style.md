# 第5章：PEP 8 とスタイル（フォーマッター）

**この章の目標**  
Python の**コーディング規約**（PEP 8）と、コードを自動で整える**フォーマッター**の使い方を学びます。  
初心者向けに、最低限押さえておくポイントを説明します。

---

## PEP 8 とは？

**PEP 8** は、Python の**公式スタイルガイド**です。

- インデント、行の長さ、命名規則などが定められている  
- 守ることで、**読みやすいコード**になり、チーム開発でも迷いにくい  
- 全部覚えなくても、「こう書くことが推奨されている」と知っておくとよい  

---

## 最低限押さえたい PEP 8 のルール

| 項目 | ルール |
|------|--------|
| **インデント** | 4スペースを使う |
| **1行の長さ** | 79文字以内が推奨（長い場合は改行） |
| **変数・関数名** | 小文字とアンダースコア（`user_name`） |
| **クラス名** | 大文字始まり（`UserService`） |
| **定数** | 大文字とアンダースコア（`MAX_COUNT`） |
| **空行** | 関数やクラスの間に2行、クラス内で1行 |

---

## フォーマッターとは？

**フォーマッター**は、コードを**自動で整えてくれるツール**です。

- インデントや改行を、PEP 8 に沿って直してくれる  
- 手動で直す手間が減る  
- **Black** や **Ruff** がよく使われる  

---

## Black の使い方

**Black** は、Python の代表的なフォーマッターです。

```bash
# インストール
pip install black

# ファイルを整形する
black example.py

# フォルダ内の .py ファイルを一括整形
black .
```

- 実行すると、その場でファイルが書き換えられる  
- 保存前に自動で整形したい場合は、エディタの「保存時に Black を実行」を有効にする  

---

## Ruff の使い方

**Ruff** は、**Linter**（問題を指摘する）と**フォーマッター**の両方の機能を持つツールです。

```bash
# インストール
pip install ruff

# フォーマット（Black 互換）
ruff format example.py

# リント（問題を指摘）
ruff check example.py
```

- **format** で整形、**check** でバグの兆候やスタイル違反を検出  

---

## まとめ

- **PEP 8** は Python の公式スタイルガイド。インデント4スペース、命名規則などを守る。
- **Black** や **Ruff** でコードを自動整形できる。
- プロジェクトでは、どれか1つフォーマッターを決めて使うと、スタイルが揃う。
