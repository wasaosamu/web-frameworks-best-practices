## Python の構文・慣習（覚えておくべきポイント）

- **コロン（:）**  
  if / for / while / def / class の行末に必ず付ける。次の行がブロックの始まりを示す。  
  例：`if x > 0:`、`def foo():`

- **インデントでブロック**  
  他の言語では `{ }` を使うが、Python は**字下げ**でブロックを表す。`{ }` は使わない。

- **None**  
  「値が無い」を表す特別な値。`null` や `nil` に相当。大文字始まり。

- **True / False**  
  真偽値。**大文字始まり**で書く（`true` / `false` ではない）。

- **pass**  
  何もしないための文。中身を後で書くときの仮置きに使う。  
  例：`def empty(): pass`

- **...（Ellipsis）**  
  `pass` の代わりや、型ヒントで「任意」を表すときに使う。`...` と書く。

- **アンダースコアの慣習**  
  `_name`：内部用（import 時に `*` で取り込まれない）。  
  `__name`：名前のマングリング（クラスで使う）。  
  `name_`：予約語との衝突を避けるため。

## Python 特有のもの

- **動的型付け**  
  変数に型を宣言しない。代入した値から自動で型が決まる。  
  型ヒントは書けるが、実行時には強制されない。

- **Duck Typing（ダックタイピング）**  
  「アヒルのように歩き、アヒルのように鳴くなら、それはアヒルだ」という考え方。  
  型ではなく「必要なメソッドを持っているか」で判断する。  
  例：`len()` は `__len__` を持っていれば使える。

- **Batteries Included**  
  Python は「電池付き」と呼ばれる。標準ライブラリが豊富で、  
  json / pathlib / re / logging など、多くの機能が最初から使える。

- **特殊メソッド（マジックメソッド）**  
  `__init__`、`__repr__`、`__len__` のように、前後にアンダースコア2つ付くメソッド。  
  特定の演算や組み込み関数に対応している。

- **リスト・辞書・集合のリテラル**  
  `[]` リスト、`{}` 辞書、`{1, 2}` 集合。  
  空の `{}` は辞書になる（空集合は `set()` を使う）。

- **スライス**  
  `[start:end:step]` でリストや文字列の一部を取り出す。  
  例：`s[1:4]`、`s[::-1]`（逆順）。

## 基本

- **変数**  
  データを入れておく箱。名前を付けて値を代入し、あとから参照できる。  
  Python では型を宣言せず、`name = "山田"` のように代入するだけ。

- **インデント**  
  行の先頭の字下げ。Python ではブロック（if / for / def / class の中身）を表す。  
  **4スペース**が PEP 8 で推奨。タブとスペースを混ぜない。

- **モジュール**  
  1つの .py ファイル。他のファイルから `import` して使える。

- **パッケージ**  
  `__init__.py` を含むフォルダ。複数のモジュールをまとめたもの。

---

## データ型

- **str**  
  文字列。`"` や `'` で囲んだテキスト。

- **int**  
  整数。

- **float**  
  小数。

- **list**  
  リスト。複数の値を順番に持つ。`[ ]` で囲む。

- **dict**  
  辞書。キーと値のペアを格納する。`{ }` で囲む。

- **bool**  
  真偽値。True か False。

---

## 関数・クラス

- **def**  
  関数を定義するキーワード。

- **return**  
  関数の戻り値を返す。

- **yield**  
  ジェネレータで、1つずつ値を返すときに使う。

- **docstring**  
  関数やクラスの説明文。`"""` で囲む。

- **class**  
  クラスを定義するキーワード。

- **インスタンス**  
  クラスから作った実体。

- **self**  
  クラスのメソッドの第1引数。そのインスタンス自身を指す。  
  慣習で `self` と書く（別名でも動くが、ほぼ常に self）。

- **__init__**  
  コンストラクタ。インスタンス作成時に呼ばれる特殊メソッド。  
  `def __init__(self, ...):` の形で定義する。

- **__repr__**  
  オブジェクトの文字列表現を返す特殊メソッド。  
  `print(obj)` やデバッグ時に使われる。dataclass では自動生成される。

- **デコレータ**  
  関数をラップして前後に処理を追加する仕組み。`@名前` で関数の上に書く。  
  例：`@app.route("/")`、`@dataclass`

- **dataclass**  
  データを保持するクラスを簡潔に書くための仕組み。  
  `__init__` や `__repr__` を自動生成する。

## 例外・エラー

- **try / except**  
  例外を捕まえて処理するための構文。

- **raise**  
  例外を発生させる。または、捕まえた例外を再送出する。

## 環境・ツール

- **venv**  
  仮想環境。プロジェクトごとにパッケージを分けて管理する仕組み。

- **requirements.txt**  
  プロジェクトで使うパッケージとバージョンの一覧。

- **PEP 8**  
  Python の公式スタイルガイド。

- **Black / Ruff**  
  コードを自動整形するフォーマッター。

- **pytest / unittest**  
  テストを書くためのライブラリ。

- **環境変数**  
  プログラムの外側で設定する値。API キーやパスワードに使う。

- **JSON**  
  データをテキストで表現する形式。API や設定ファイルでよく使う。

---

## 実務でよく使う

- **pathlib**  
  ファイルパスをオブジェクトとして扱う標準ライブラリ。Path クラスを使う。

- **正規表現**  
  文字列のパターンを表現する記法。re モジュールで使う。

- **map**  
  各要素に関数を適用して、新しいイテラブルを作る組み込み関数。

- **filter**  
  条件に合う要素だけを残す組み込み関数。

- **zip**  
  複数のリストを1つずつペアにしてまとめる組み込み関数。

---

## イテラブル・イテレータ

- **イテラブル**  
  `for` で回せるオブジェクト。リスト、辞書、文字列、range など。  
  `iter()` でイテレータを取得できる。

- **イテレータ**  
  `next()` で次の要素を返すオブジェクト。`for` の裏側で使われる。

- **range**  
  連番を作る組み込み。`range(5)` で 0, 1, 2, 3, 4。  
  メモリを節約するため、リストではなくイテラブルを返す。

---

## その他よく使う構文

- **f-string**  
  `f"こんにちは、{name} さん"` のように、変数を文字列に埋め込む書き方。  
  Python 3.6 以降で推奨。

- **三項演算子**  
  `x if 条件 else y`。条件が True なら x、False なら y を返す。

- **アンパック**  
  `a, b = [1, 2]` や `first, *rest = [1, 2, 3]` のように、まとめて代入する。  
  `*` で残りをリストとして受け取れる。

- **in**  
  `x in [1, 2, 3]` で「含まれるか」を判定。辞書ではキーの存在チェックに使う。
