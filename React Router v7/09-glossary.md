# 用語集（React Router v7）

この用語集は、本文（第1章〜第8章）で出てくる **覚えておくと読みやすくなる単語**をまとめたものです。  
「わからない単語が出たらここに戻る」ためのページとして使ってください。

---

## ルーティング

- **ルート（Route）**  
  「このURLならこの画面」という対応の1単位です。

- **path**  
  ルートが担当するURLのパスです（例：`/`、`/about`、`/users/:id`）。

- **動的セグメント（`:id`）**  
  URLの一部が変わる場所です（例：`/users/123` の `123` を `:id` で受け取る）。

- **ネストルート（Nested Routes）**  
  ルートを親子で入れ子にすることです。親はレイアウト、子は中身、という分け方ができます。

- **Outlet**  
  「子ルートの画面が差し込まれる場所」です。レイアウトの中に `<Outlet />` を置きます。

---

## データ（v7の重要ポイント）

- **loader**  
  そのルートが表示される前に動く「データ取得の関数」です。結果は `useLoaderData()` で受け取ります。

- **useLoaderData**  
  loader が返したデータを、ルートコンポーネントで受け取るためのフックです。

- **action**  
  フォーム送信など「データ変更」を受ける関数です。`request.formData()` で入力を受け取ります。

- **Form**（`<Form>`）  
  React Router のフォームです。`method="post"` で送ると、該当ルートの action が呼ばれます。

- **useActionData**  
  action の戻り値（エラーメッセージ等）をコンポーネント側で受け取るためのフックです。

- **redirect**  
  action 内で「成功したらこのURLへ移動してね」と伝える仕組みです。

---

## ナビゲーション

- **Link**（`<Link>`）  
  アプリ内遷移に使うリンクです。`<a href>` より基本はこちらを使います。

- **useNavigate**  
  プログラムから移動したいときに使います（例：戻る、成功後に移動）。

- **useSearchParams**  
  URLのクエリ（`?page=2` など）を読み書きするためのフックです。

- **useNavigation**  
  いま「遷移中か」「送信中か」を教えてくれます。ローディング表示や二重送信防止に使います。

---

## エラー

- **errorElement**  
  そのルート（と子）でエラーが起きたときに表示するコンポーネントです。

- **useRouteError / isRouteErrorResponse**  
  errorElement の中で、投げられたエラーの中身を取り出して表示するための道具です。

- **`throw new Response(...)`**  
  loader / action で「404」などをエラーとして扱いたいときの書き方です。
